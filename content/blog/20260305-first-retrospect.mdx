## 왜 리뉴얼했나

기존 큐니버스는 우주 테마 기반의 정적 포트폴리오였다. 충분히 괜찮았지만 두 가지 한계가 있었다:

1. **콘텐츠 확장이 어려움** — 새로운 섹션을 추가하려면 레이아웃부터 다시 짜야 했다
2. **차별화 부족** — 우주 테마 포트폴리오는 이미 많다

## Neo Brutalism을 선택한 이유

- 두꺼운 보더, 강한 그림자, 선명한 색상 — 한눈에 시선을 잡는다
- 윈도우 매니저 UI와 궁합이 좋다 (데스크탑 메타포)
- 직접 만든 디자인 키트(`@kokyusik91/neo-brutal-ui`)를 실전에서 쓸 수 있다

## 잘한 것

- **컴포넌트 설계** — Window, Desktop, Taskbar를 분리한 구조가 확장에 유리했다
- **디자인 토큰 통일** — `neo-border`, `neo-surface`, `shadow-neo-md` 같은 토큰으로 일관된 스타일 유지
- **드래그 앤 드롭** — `useDrag` 훅 하나로 아이콘/윈도우 드래그를 모두 처리

## 아쉬운 것

- 모바일 대응이 아직 미흡하다. 데스크탑 UI 자체가 모바일에 맞지 않는 메타포라 고민이 필요하다
- 접근성(a11y) 체크를 초반에 안 해서 나중에 한꺼번에 잡아야 한다

## 배운 점

> 완벽한 설계를 기다리지 말고, 동작하는 것부터 만들자. 구조는 만들면서 잡힌다.

처음에 윈도우 매니저 상태 관리를 어떻게 할지 오래 고민했는데, 결국 단순한 `useState` + Context로 시작해서 필요할 때 개선하는 게 훨씬 빨랐다.
