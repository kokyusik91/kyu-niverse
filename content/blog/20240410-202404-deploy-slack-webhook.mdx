## pipeline 페이지를 새로고침하고 있었다

코인니스 웹과 어드민 프로젝트는 GitLab CI/CD로 빌드 및 배포를 하고 있다. `dev`, `stg`, `master` 브랜치에 머지하면 자동으로 배포가 된다. 그리고 개발자가 배포 완료 후 반영 내용을 정리해서 슬랙 채널에 공지해야 한다.

문제는 배포가 됐는지 확인하는 과정이었다.

1. GitLab에서 Merge 버튼을 누른다
2. 기다린다…
3. GitLab pipeline 페이지에 들어가서 체크 표시가 떨어졌는지 확인한다
4. 배포 완료를 확인하면 반영 내용을 정리해서 채널에 공지한다

pipeline 페이지를 새로고침하면서 기다리는 게 매번 번거로웠다. 배포가 완료되면 알아서 알림이 오면 좋겠다고 생각했다.

## Slack Incoming Webhook

회사에서 전사적으로 슬랙을 쓰고 있었기 때문에, Slack의 Incoming Webhook을 사용하면 된다는 걸 이전에 웹훅을 공부하면서 알고 있었다. Incoming Webhook이란 외부에서 특정 슬랙 채널로 메시지를 보내는 기능이다. HTTP POST 요청으로 메시지를 payload에 담아 보내면 된다.

`coinness-admin-deploy`라는 채널을 만들고 Incoming Webhook을 연결했다. 슬랙에서 웹훅 URL을 제공해준다.

## webhook.sh — 관심사 분리

메시지를 보내는 시점은 "각 환경에 배포가 완료되었을 때"다. `.gitlab-ci.yml` 파일에 직접 웹훅 코드를 넣을 수도 있었지만, 그러면 관심사 분리가 안 되고 수정할 때마다 yml 파일을 건드려야 한다.

그래서 `webhook.sh`를 별도 파일로 만들고, `.gitlab-ci.yml`에서는 "배포 완료 시 이 스크립트를 실행해라"만 지정하도록 했다.

![배포-완료시-슬랙-훅으로-완료됬다는-메시지를-받게하자-04.png](/blog/images/20240410-202404-deploy-slack-webhook-01.png)

```shell
# webhook.sh
send_slack_message() {
    local slack_url="$1"
    local environment="$2"
    local status="$3"
    local current_time=$(TZ=Asia/Seoul date +"%Y-%m-%d %H:%M:%S")
    local message="$environment 서버에 배포가 $status 하였습니다. 📣 - 현재 시간: $current_time"
    curl -X POST --data-urlencode "payload={\"text\": \"$message\", \"icon_emoji\": \":ghost:\"}" $slack_url
}
send_slack_message $1 $2 $3
```

`$1`에 웹훅 URL, `$2`에 환경 이름(dev, stg, production), `$3`에 성공/실패 여부가 들어간다. 환경과 상태를 동적으로 받으니까 하나의 스크립트로 모든 환경에 대응할 수 있다.

`.gitlab-ci.yml`에서는 이렇게 호출한다.

```yaml
after_script:
  # 성공했을 때
  - if [ $CI_JOB_STATUS == "success" ]; then bash ./webhook.sh $DEPLOY_SLACK_WEBHOOK_URL Production 성공; fi
  # 실패했을 때
  - if [ $CI_JOB_STATUS != "success" ]; then bash ./webhook.sh $DEPLOY_SLACK_WEBHOOK_URL Production 실패; fi
```

웹훅 URL은 코드에 직접 넣지 않고 GitLab의 CI/CD Variables에 `DEPLOY_SLACK_WEBHOOK_URL`로 등록해서 환경변수로 관리했다.

## 시간이 한국 시간으로 안 찍힌다

첫 시도에서 배포 알림이 잘 왔다. 근데 메시지를 자세히 보니 시간이 한국 시간이 아니었다.

![배포-완료시-슬랙-훅으로-완료됬다는-메시지를-받게하자-05.png](/blog/images/20240410-202404-deploy-slack-webhook-02.png)

GitLab CI Runner의 시스템 시간이 UTC 기준이라 그런 거였다. 쉘 스크립트에서 `TZ=Asia/Seoul`을 명시해줘야 한다.

```shell
TIME=$(TZ=Asia/Seoul date +"%Y-%m-%d %H:%M:%S")
```

## 팀에 퍼지다

이 기능을 도입하고 나서 pipeline 페이지에 들어가서 배포 유무를 확인할 필요가 없어졌다. 알림이 오면 반영 내용만 정리해서 공지하면 끝이다.

팀원들을 배포 알리미 채널에 초대했더니 반응이 좋았다. 처음에는 어드민에만 붙였는데, 쓰다 보니 이 기능이 없는 코인니스 웹 서비스 쪽에서 불편함이 더 크게 느껴졌다. 팀원들이 웹 서비스에도 붙여달라고 요청하셨다.

![배포-완료시-슬랙-훅으로-완료됬다는-메시지를-받게하자-02.png](/blog/images/20240410-202404-deploy-slack-webhook-03.png)

프론트엔드에 국한되지 않은 이런 프로그래밍은 재밌다. 외부 서비스에 연동해서 특정 이벤트를 발생시키는 건 도파민이 나오기에 충분했다. 간단한 웹훅 기술 하나로, 팀 전체의 배포 프로세스가 좀 더 편해졌다.
